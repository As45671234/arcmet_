# Инструкция по обновлению сервера (VPS)

Это пошаговая инструкция для загрузки последних изменений из GitHub на ваш рабочий сервер.

## Шаг 1: Подключение к серверу
Откройте терминал (или командную строку) и подключитесь к серверу по SSH:
```bash
ssh root@<ВАШ_IP_АДРЕС_СЕРВЕРА>
```
*(Если вы используете другого пользователя, например ubuntu, то пишите `ssh ubuntu@...`)*

## Шаг 2: Переход в папку проекта
Перейдите в директорию, где у вас хранится проект:
```bash
cd /путь/к/вашему/проекту
```
*(Например: `cd /var/www/arcmet` или `cd ~/arcmet`)*

## Шаг 3: Получение новых изменений
Скачайте последние изменения с ветки `main`:
```bash
git pull origin main
```

## Шаг 4: Обновление Backend (Серверная часть)
Перейдите в папку backend, обновите зависимости и перезапустите сервер через PM2:
```bash
cd backend
npm install
pm2 restart all
```

## Шаг 5: Обновление Frontend (Пользовательская часть)
Вернитесь в начало проекта, перейдите в папку frontend, обновите зависимости и соберите заново билд:
```bash
cd ../frontend
npm install
npm run build
```

*(Если ваш Nginx просто раздает папку `dist`, дополнительных действий не нужно. Если требуется применить новые настройки Nginx, выполните: `sudo systemctl restart nginx`)*
